Tři instrumenty v `feat/terrain-perf`:

**Baseline (po material cache, před atlasem) @ 30×30:**

| metrika | hodnota |
|---|---|
| FPS | 15 |
| calls | 25 106 |
| tri/call | ~2 (smoking gun) |
| generateTerrain | 2.5 ms |
| spawnTerrain.blocks | 38.6 ms |
| total regen | ~43 ms |
| `[Violation]` rAF handler | 50-64 ms |

Klíčový nález: `tri/call ≈ 2` = 1 face per draw call. To je `THREE.Mesh` s `material = [m0..m5]` rozsekáním na 6 material groups → **6× draw calls per box**. Material cache šetří GPU state changes mezi calls, ale počet calls neredukuje. Generace = trivial (5 % regen), worker dead. Render loop = jasný bottleneck.

**Atlas refactor:**
- `getSharedAtlasBoxGeom()` — sdílená `BoxGeometry` (1 instance pro všechny terrain TCUBES) s UVs přepsanými na 1/6-tice atlasu (face N → u ∈ [N/6, (N+1)/6]).
- `getTcubesKindMaterial(kind)` — per-kind atlas: 6 facelets slepené horizontálně do CanvasTexture 96×16 px, `NearestFilter` + `SRGBColorSpace`. Negative cache pro non-terrain kindy.
- `createTCubeFor` fast/slow path dispatch: terrain kind (matches `BLOCK_TEXTURES`) → shared geom + atlas material (1 draw call). Jiné → původní material array (slow path zachován pro emoji demo boxy).

**Po atlas refactoru @ 30×30:**

| metrika | před atlasem | po atlasu | změna |
|---|---|---|---|
| FPS | 15 | **92** | **6.1×** |
| calls | 25 106 | ~5 050 | 0.20× |
| tri | 49 432 | 49 480 | shoda |
| geom | ~5 500 | **8** | sdílená box geom |
| spawnTerrain.blocks | 38.6 ms | 6.5 ms | 5.9× rychlejší |
| total regen | 43 ms | 9 ms | 4.8× rychlejší |
