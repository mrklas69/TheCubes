### Tick model

- **1 wall sekunda = 1 tick** (globální `TIME.tick`, dosud DD-04 neused — konečně získává konzumenta).
- `productionTick(dt)` v render loopu, ne per-1s setInterval. `dt = wall second elapsed × world.TIME_SCALE`. Clamping `dt ≤ 0.1 s` (proti tab-sleep spikes).
- Rate jako float ks/s (les 0.5/s = 30/min), nikoli integer per-tick.
- Resource unit MVP = abstraktní „ks" napříč (logs, water, …). Voidspan `formatScalar` (l/m³/kg) odložen na Phase 2.

### Event Log model

- Voidspan-derived: 4-znakové verbs (MVP: `PROD`, `DRN`, `PAUS`, `RSUM`).
- Ring buffer kapacita 100, viditelných 5 v `#eventlog` (left bottom).
- **Emit jen na state change**, ne per-frame:
  - `PROD` / `DRN` — jen na **floor crossing** (whole-unit produkce / spotřeba). Per-frame produkce 60× by zahltila log.
  - `PAUS` / `RSUM` — jen na change `facility.PAUSED` true↔false.
- Severity-colored CSS: PROD zelená, DRN neutrální šedá, PAUS amber, RSUM cyan.

### Implementační detaily

**Atributy `PATH` rozšíření:**
```js
this.SOURCE     = null;   // instance ref na FACILITY
this.SINK       = null;
this.RESOURCE   = null;   // string klíč do RESOURCES_DEF
this.THROUGHPUT = null;   // ks/s; null = decorative
```

Defaultní `null` zachovává backward compat se sez. 17 dekorativní `path_0` (severská cesta z tunelu). `pathTick` přeskočí missing-fields checkem — žádné větvení decorative vs. factory na úrovni registry.

**`pathTick(dt)` v render loopu** se volá **po** `productionTick(dt)` — pořadí důležité: tick = (1) generator/transformer produkce, (2) PATH distribuce. Opačné pořadí by transport tahal items, které generator ten samý tick ještě nevyrobil.

**`transferOnPath(path, dt)`** — jediná transport jednotka:
```js
const have = src.BUFFER[r] ?? 0;
const free = sinkDef.buffer_capacity - (dst.BUFFER[r] ?? 0);
const amount = Math.min(path.THROUGHPUT * dt, have, free);
src.BUFFER[r] -= amount;  dst.BUFFER[r] += amount;
maybeLogConsumption(src, r, ...);  maybeLogProduction(dst, r, ...);
```

Reuse existing floor-crossing helperů znamená, že DRN/PROD eventy z PATH transportu mají stejnou granularitu (1 event per whole-unit krok) jako z productionTick — bez per-frame spamu.

**Boot-time validace category × KIND** v `createPathFor`: pokud `RESOURCES_DEF[RESOURCE].category === "fluid"` & `KIND !== "pipeline"`, console.warn (analog pro solid/conveyor). Nepostihne dekorativní paths (`RESOURCE === null`). Strict reject by byl příliš tvrdý — KISS warn stačí pro development.

Range 0..3 step 0.1, accent-color cyan (sladěno s `evt-rsum` cyan). `input` event listener mutuje `world.TIME_SCALE` + aktualizuje text `1.0×` label. Render loop si TIME_SCALE čte každý frame → změna se projeví okamžitě bez reloadu.
