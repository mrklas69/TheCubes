## Sezení 12 — Humanoidní varianty: NOODLE + STICKMAN, DD-20 poseFns dispatch

### Diskuse

Cíl: zkusit alternativní tvary postavičky jako experiment vs. hinge-based CHARACTER. Dva typy:

1. **NOODLE** — „plastelínový poutník": dvě capsule-fazole (tělo + hlava) a čtyři tube-makaron končetiny podél `CatmullRomCurve3`. Myšlenka: per-frame mutace kontrolních bodů křivky → regenerace `TubeGeometry` → organický plastelínový ohyb místo rigidního hinge.

2. **STICKMAN** — uživatelem dodaná formální spec s obrázkem (low-poly blokový styl: kvádr trup, low-poly koule hlava, 6-segmentové válce končetin, kostkové ruce/chodidla, „plná kloubová struktura"). Geometricky je to jiná verze CHARACTER — sdílí hinge pipeline, jen s blokovou estetikou a 3 segmenty per končetina (přidané zápěstí a kotník).

**Konceptuální jádro sezení:** obě varianty narazily na otázku *„jak sdílet animační pipeline mezi různými tvary postaviček?"*. NOODLE má úplně jinou topologii (Tube podél křivky, ne kloubové rotace) → potřebuje vlastní walkCycle. STICKMAN má stejnou topologii jako CHARACTER + navíc terminal klouby → stejný walkCycle s volitelným rozšířením.

Řešení: **`poseFns` dispatch** v `group.userData`. Každá build fn naplní `userData.poseFns = { walkCycle, sitPose, liePose, workPose, reset }` — animátory `walk`/`sit`/`wander`/`enterWanderState` pak dispatchují přes tuto mapu, ne přímo. CHARACTER poseFns ukazuje na existující primitives, NOODLE má vlastní curve-based implementaci, STICKMAN re-usne CHARACTER (díky rozšíření applyWalkCycle o `if (p.leftWrist)` backward compat check).

**Zásadní momenty:**

- **Plán → Go! pattern proběhl čistě.** NOODLE: 5 otázek plánu (rozsah, živost křivek, ctrl body, dvě fazole?, nová třída?) → krátké odpovědi uživatele (`A1 modrý / A2 živé / A3 3 / A4 2 / A5 nová`) → přímá implementace, 6 tasků lineárně. STICKMAN: 4 otázky (sdílet poseFns?, animovat terminal klouby?, pozice?, obličej?) → `A1 ano / A2 ano / A3 a / A4 ne` → 4 tasky.
- **Iterativní doladění NOODLE přes user fixy.** První verze byla moc volná (ruce visí jak klasové řady), uživatel poslal dvě vlny fixů:
  - **První vlna (F1–F4):** hlava zabořená do body (60 % headR uvnitř), průměry trubek na polovinu (armR 0.06→0.03, legR 0.08→0.04), nohy zabořené do spodní polokoule body, paže pod 45° ven-dolů (armOffset = armLen / √2).
  - **Druhá vlna (F1–F2):** ramena posunuta výš (z cylinder středu 0.47 do horní polokoule 0.60), přidán loket = `buildLimb` refactor na variabilní počet uzlů (paže 4 uzly: rameno-loket-zápěstí-ruka s forward Z offsetem pro pokrčený rest pose; nohy 3 uzly). Pose primitives iterují přes `limb.currentCtrl.length` s lineárním shift factorem `i/(n-1)`.
- **A1+A2 konflikt u STICKMANa.** Uživatel řekl „ano sdílet poseFns" **a** „ano animovat wrist/ankle" — zdánlivě rozporné. Interpretace: sdílet **dispatch klíče** (walkCycle/sitPose/atd.), ale `applyWalkCycle` rozšířit o **volitelné** terminal klouby přes `if (p.leftWrist)` check. CHARACTER bez wristů = backward compat, STICKMAN s wristy = dodatečná rotace `extra2 = lowerFactor × 0.5` (overlapping action). Sit pose ankle +π/2 kompenzuje ohyb kolena −π/2 → chodidlo zůstane horizontálně. Bez nutnosti parallel walk cycle.
- **Per-frame TubeGeometry rebuild.** NOODLE walk cycle dispose+new `TubeGeometry` × 4 končetiny × 60 Hz = 240 GPU uploadů/s. Idempotentní `resetNoodleLimbs` přeskočí rebuild pokud ctrl body už jsou v rest → šetří zbytečné uploady při statických stavech (stand/sit). Tubular segmenty drženy nízko (10–12), radial 6 → každý upload ~pár KB, celkově zanedbatelné.
- **Rozdíl v přístupu k „sdílení".** NOODLE = nová ANIMATE implementace s vlastními pose primitives (paralelní k CHARACTER). STICKMAN = rozšíření existujících primitives. Obojí pod stejným poseFns slotem, obojí pod stejným ANIMATE.kind ("walk", "sit", "wander") — jediný user-facing kontrakt se nerozbil.

### Schválená rozhodnutí

**DD-20 — Humanoidní varianty sdílejí `ANIMATE` mode slot přes `poseFns` dispatch.** Každá humanoidní třída (CHARACTER, NOODLE, STICKMAN, …) v build fn naplní `group.userData.poseFns = { walkCycle, sitPose, liePose, workPose, reset }`. Animátory `walk`/`sit`/`wander` a `enterWanderState` dispatchují přes tuto mapu; pose primitives mohou být sdílené (CHARACTER ↔ STICKMAN) nebo vlastní (NOODLE s curve-based). Nová varianta = nová build fn + poseFns, žádný zásah do animátorů.

**Kandidát DD-21** (přečíslováno z původního DD-20 kandidáta): `STATE` atribut pro CHARACTER — wander substate visibility v infotipu.

### Kód

**`src/model.js`:**

- Nové třídy `NOODLE extends COMPOSITES` a `STICKMAN extends COMPOSITES`, obě s atributem `COLOR`. Docstring popisuje geometrii + podporované ANIMATE.kind + sdílení poseFns.

**`src/main.js`:**

- Import rozšířen o `NOODLE`, `STICKMAN`.
- **`buildNoodle(group, instance)`** — 2 CapsuleGeometry (tělo + hlava, hlava zabořená `headR * 0.4`), 4 TubeGeometry končetiny přes `buildLimb(points, radius, swingSign)` (variabilní počet uzlů). Paže 4 uzly s forward Z offsetem na loketu (−0.06) a zápěstí (−0.03), nohy 3 uzly. `group.userData.poseFns` = `{ applyNoodleWalkCycle, applyNoodleSitPose, applyLiePose, applyNoodleWorkPose, resetNoodleBase }`.
- **`rebuildNoodleLimb(limb)`** — regenerace TubeGeometry z aktuální `limb.curve` (sdílí `points` referenci s `limb.currentCtrl`). Dispose povinný.
- **`applyNoodleWalkCycle`** — iteruje přes n kontrolních bodů s `factor = i/(n-1)` → start stojí, každý další uzel progresivní Z shift, end plný `amp * length * swing`.
- **`resetNoodleLimbs`** — idempotent reset (skip rebuild pokud už v rest).
- **`buildStickman(group, instance)`** — BoxGeometry trup 0.4×0.5×0.24, 8×4 SphereGeometry hlava, `buildStickLimb` 3-segmentová (upper/lower/terminal Group hierarchie s hinge pivoty). Chodidla vysunutá dopředu (−Z) s přesně dopočítaným offsetem pro Y=−0.5 na groundplane. Sdílí poseFns s CHARACTER přes reference na `applyWalkCycle`/`applySitPose`/atd.
- **`applyWalkCycle`** rozšířen o volitelnou animaci `leftWrist`/`rightWrist`/`leftAnkle`/`rightAnkle` s amplitudou `lowerFactor × 0.5` (overlapping action). CHARACTER zůstane backward compat (bez wristů = skip).
- **`applySitPose`** — wristy na 0, kotník +π/2 (kompenzace kolena) → chodidlo horizontálně.
- **`applyWorkPose`**, **`resetCharBase`** — wristy/kotníky na 0 pokud existují.
- **`animateWalk`, `animateSit`, `enterWanderState`, `animateWander`** refaktorovány na dispatch přes `group.userData.poseFns.X(...)` místo přímého volání.
- **`buildCharacter`** na konci naplní `group.userData.poseFns` s CHARACTER primitives.
- **Dispatch:** `createCompositeFor` větve pro NOODLE + STICKMAN; `COLLISION_RADII.NOODLE = 0.2`, `COLLISION_RADII.STICKMAN = 0.2`.
- **Instance:** `char_0001` přehozen z CHARACTER na NOODLE (modrý poutník, pozice 2, 0, -2); `char_0004` nový STICKMAN červený na (1, 0, 2.5), wander subjects = [rock1, tree1].
