**Pak pivot na MagicaVoxel pipeline.** Velký koncepční moment:

- **První import:** uživatel udělal 2 auta v MagicaVoxelu, exportoval jako
  `cars-0.obj`/`cars-1.obj`. Implementoval jsem `VOXEL_MODEL` třídu (potomek

  OBJLoader+MTLLoader z `./assets/`, auto-center XZ + bottom snap Y,
  NearestFilter pro pixel-art look). „Kudos! Jsou tam!" — první external
  asset workflow funguje.
- **Reverse pipeline:** uživatel chtěl exportovat naši grass cube DO MagicaVoxelu.
  Napsán Node skript `tools/export-grass-vox.mjs` → `assets/grass-cube.vox`

- **Tunel z MagicaVoxelu:** uživatel poslal `tunel.obj`. Bbox 1.6×1.6×1.6
  (= 16³ MV voxelů), scale 0.625 = 1 TheCubes voxel. Replaced TUNNEL_ARCH
  half-torus instances. Problém: **paleta má jen 1 barvu** (#5d9446 = naše
  grass). Model je celý zelený. User: „obarvit, dokončit?" — vysvětlil jsem
  že barvy musí přidat v MagicaVoxelu. Vygeneroval jsem `assets/scene-palette.vox`
  jako šablonu s 12 swatch barvami (4 grass + 4 dirt + 4 stone) v 4×3 mřížce
  v paletě + reference RGB tabulka.
- **Simplifikovaná auta:** uživatel: „uměl bys zmenšit/zjednodušit obě auta?"
  Generoval jsem `assets/car-simple-{0,1}.vox` (sedan modré + truck červený,
  ~55 voxelů každé) — 7×3×4 layout: 4 kola dole, body, sklo (čelní + boční),
  střecha, žluté světlomety. User otevře v MagicaVoxelu, ověří, exportuje.

**Shadow problem & fix.** User popsal: „edge kostky" mají tenkou světelnou
linku zdola/zprava. Diagnóza: **klasický peter-panning** — `normalBias = 0.02`
posouval shadow sample mimo skutečný povrch, na styku voxelových ploch
zanechával lit line. Iterace: 0.02 → 0.005 (pomohlo částečně) → 0 + bias
−0.0005 → −0.0001 (2/3 pryč) → `PCFSoftShadowMap` → `PCFShadowMap` (sharper
= bez blur leak). **„Takto to stačí."** Trade-off: stíny ostřejší, méně
soft, ale pro voxel scénu vypadá konzistentně s pixel-art.

**Ujasnění vizuálních zdrojů.** User: „Z jaké grafiky budeme stavět TheCubes
scény?" Diskuse 4 zdrojů (procedurální COMPOSITES, procedurální canvas
textury, canvas SPRITES, externí 3D modely). Q1=B (hybrid: parametrized vs
static), Q2=mix dle účelu, Q3=ano (VOXEL_MODEL default pro statickou
geometrii). **Zafixováno DD-21**, GLOSSARY rozšířen o sekci „Vizuální zdroje"
s tabulkou workflow rozhodnutí, CLAUDE.md Status + hierarchie aktualizovány
(4 nové třídy: TUNNEL_ARCH, WAREHOUSE, TRAIN, VOXEL_MODEL).

### Schválená rozhodnutí

- **DD-21 — Vizuální zdroje: hybrid (procedurální COMPOSITES + externí
  VOXEL_MODEL).** Parametrizované entity (atributy ovlivňují vzhled, animátor
  sahá na parts) → procedurální COMPOSITES. Statická dekorace → VOXEL_MODEL
  z MagicaVoxelu (default pro novou statickou geometrii). Voxelová podlaha
  → TCUBES + `:named-texture`. Dialog/UI → SPRITES. Hybrid maximalizuje silné
  stránky každého zdroje. Pixel-art styl (NearestFilter + 16×16) zaručuje
  vizuální konzistenci napříč zdroji.
