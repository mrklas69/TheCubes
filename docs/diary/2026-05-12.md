5 reálných rizik (od největšího):
1. **File-level race** v pracovním stromu — dvě instance editují stejný soubor, druhý zápis přepíše první (žádný OS lock, Edit tool chytí jen předchozí Read téže instance).

3. **Docs race** — DIARY.md / TODO.md / MEMORY.md (index) — Write přepíše bez merge.
4. **Session numbering collision** — obě instance napíšou `## Sezení 24` v diáři.
5. **Port 8000** — bezpečné (jeden listener, ostatní vidí „už poslouchá").

Doporučení (od silného ke kompromisu):
- **`git worktree add ../TheCubes-<topic> feat/<topic>`** per instance — každá instance vlastní adresář + větev, konflikty se řeší až mergem. Nejčistší.
- **„Dirigent" pattern** — jedna instance smí docs + `%BEGIN`/`%END` + commits, ostatní jen kód v izolovaném souboru (např. `src/terrain.js` ve sdíleném worktree, kde nikdo jiný `src/` needitujе).
