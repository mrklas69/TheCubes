## Sezení 8 — ROCK, dynamický 3D ocásek SPRITES, audit fixy

### Diskuse

Otevření v prázdné pracovní složce — repo `TheCubes` smazáno, uživatel požádal
o `obnov https://github.com/mrklas69/TheCubes`. Klon + průzkum stavu (M1–M7
hotové, M8+ průběžně: HOUSE, CLOUD + `drift`, `pulse`, `rotate`, `orbit_stadium`).

Na otázku „co dál" AI předložila tři kandidáty z TODO/IDEAS: **(1) ROCK**
jako rozcvičku, **(2) dynamický 3D ocásek SPRITES** jako malý koncept-krok,
**(3) rampy / graf pohybu** jako nejhlubší téma. Uživatel zvolil **1+2**.

Po implementaci vyvolán `%AUDIT:CODE` — šest findings (F1–F6). Uživatel
požádal „oprav všech 6".

**Zásadní momenty:**

- **SPEAKER dispatch** — nový duck-typing pattern na tvaru hodnoty (instance
  vs. literál), izomorfně s DD-14 (vizuální atributy) a DD-15 (`ANIMATE`
  recepty). Formalizováno jako DD-16. `SPEAKER_OFFSET_Y` jako samostatný
  atribut (ne vnoření do SPEAKER) — drží SPEAKER slot jednoduchý (instance
  nebo `{x,y,z}`), offset je nezávislý ladicí parametr.
- **Derived behavior vs. user recipe** — bubble tail je engine-interní
  důsledek vyplněného `SPEAKER` (ne user-deklarovaný recept jako `ANIMATE`).
  Druhý per-frame registry (`bubbleTails` vedle `animators`) to odráží.
  Pokud přibude třetí derived behavior, refaktor na obecný `updaters[]`
  (zatím zafixováno komentářem v kódu i DD-16).
- **Mesh lookup pro tracking** — `meshByInstance` mapa řeší, že animátory
  jako `orbit_stadium` mutují `object3d.position`, ne `instance.X/Y/Z`.
  Ocásek tak sleduje orbitující krabici live, ne statickou polohu modelu.
- **Audit finding F1** (magic offsetY +1.5) — fixní default nepasoval oběma
  cílům (tree chtěl 1.8, voxel 0.5). Řešení: `SPEAKER_OFFSET_Y` atribut
  s defaultem 0.5 (voxel vrch), TREE přepsán ručně.
- **Canvas textura bubliny** zjednodušena — ocásek odstraněn z canvasu,
  plátno nyní čistý zaoblený obdélník 512×150. `bubbleFraction` odstraněn
  ze zpětné hodnoty, `spriteHeight` fixně 0.5.

### Schválená rozhodnutí

- **DD-16** — `SPRITES.SPEAKER` dispatch podle tvaru hodnoty: instance
  (`.X/.Y/.Z` + `meshByInstance` lookup + `SPEAKER_OFFSET_Y`) vs. literál
  (`.x/.y/.z`). Formalizuje engine-interní „derived behavior" kategorii
  (odlišné od user-deklarovaného `ANIMATE`, DD-15). Precedent pro nový
  engine registry; pokud přibude třetí, refaktor na `updaters[]`.

### Kód

**Model (`src/model.js`):**

- `class ROCK extends COMPOSITES` — atribut `COLOR` (default `0x808080`
  šedá). Dokumentace zdůrazňuje uzavření pětice COMPOSITES (organický /
  mechanický / stavební / atmosférický / geologický).
- `SPRITES` constructor inicializuje dva nové atributy: `SPEAKER = null`
  a `SPEAKER_OFFSET_Y = 0.5`. Vyplňují se post-hoc stejně jako `ANIMATE`.

**Engine (`src/main.js`):**

- `import` přidal `ROCK`.
- `makeBubbleTexture` — odstraněn ocásek z canvasu (plátno 512×150, čistý
  zaoblený obdélník + text). Vrací `{ texture, aspect }` bez `bubbleFraction`.
- `buildBubbleTail()` — `ConeGeometry(0.06, 1, 4)`, bílý `MeshStandardMaterial`.
  Unit-height (scale.y = skutečná délka), `userData.noShadow = true` flag.
- `bubbleTails[]` registry + `meshByInstance` Map (instance.ID → Object3D)
  pro tracking world position.
- `resolveSpeakerTarget(speaker, offsetY, out)` — duck-typing dispatch
  (`.X` instance s mesh lookup / fallback, vs. `.x` literál).
- `updateBubbleTail(entry)` — per-frame: start = bubble.center + dir ×
  bubbleHalfHeight (okraj bubliny), tip = target, orientace přes
  quaternion `_up → _dir` (stejný pattern jako `cylinderBetween`).
- `updateBubbleTails()` v render loopu **po** `updateAnimations` (aby
  četl post-animator `object3d.position`).
- `createSpriteFor` — pokud `SPEAKER` vyplněn, vrátí `Group` obsahující
  sprite + tail (Group je identity transform, takže `sprite.position`
  zůstává world). Bez `SPEAKER` zůstává chování předchozí (sprite sám).
- `createMeshFor` — plní `meshByInstance`. Traverze castShadow respektuje
  `userData.noShadow` (opt-out pro vizuální indikátory).
- `createCompositeFor` — ROCK dispatch: `buildRock(group, instance)`.
- `buildRock(group, instance)` — paleta 3 odstínů (×1 / ×0.75 / ×1.2)
  se **sdílenými materiály** (3 materiály pro 5 mesh-ů). Puffs array
  `[x, y, z, radius, materialIndex]`, deterministické rotace per index.
  `flatShading: true` → ostré facety (idiom tesaného kamene).
- Nové instance: `rock_0001` (-3, 0, -2) pískovcová šedohnědá; `dialog_0001`
  přesunut na (4.6, 2.7, 1.2) mimo osu stromu + `SPEAKER = tree1` +
  `SPEAKER_OFFSET_Y = 1.8`; nový `dialog_0002` nad orbit-dráhou krabice
  (-3, 3.2, 2) + `SPEAKER = tbox2` (default offset 0.5 = vrch voxelu).

## Sezení 9 — TIMER + BALLOON lantern + COUNTER + edge highlight

### Diskuse

Uživatel zvolil **nevizuální OBJECTS → TIMER** z Příště sez. 8. Po nabídce
abstraktních kandidátů (COUNTER v HUD, TIMER+barva, RULE framework, kombo)
přidal vlastní zábavný twist: **balón jako zapínatelný zdroj světla** —
buď časovaně nebo kliknutím. Textura se rozzáří, okolí dostane druhou
sadu stínů.

AI předložila architektonický náčrt (model: `BALLOON.LIT` + `TIMER.ACTION`,
engine: PointLight, fade watcher, click handler, `tickHandlers[]` +
`ACTIONS` dispatch) a tři otevřené volby:

1. **LIT bool** vs. `EMISSIVE_INTENSITY` float — uživatel zvolil **bool**.
2. **Instant toggle** vs. **fade lerp** — uživatel zvolil **fade**.
3. Click + TIMER oba, nebo jen jeden — uživatel zvolil **oba**.

Implementace proběhla napoprvé bez ladění (→ Kudos).

**Zásadní momenty:**

- **DD-17 kategorizace:** `ANIMATE` (DD-15) = plynulé per-frame chování,
  `TIMER.ACTION` = diskrétní event-driven reakce. Jsou komplementární,
  ne konkurenční — ANIMATE patří na hostitelskou instanci (recept
  chování), TIMER je samostatná entita (recept události).
- **Dvě cesty, jeden stav:** TIMER i click handler mění stejný
  `balloon.LIT` bool. Stav žije v modelu, dispatch mechanismy jsou jen
  „ovladače". Fade watcher (engine-derived, izomorfní s DD-16 bubble
  tail) vidí jen výsledek, nezajímá ho, kdo stav změnil.
- **PointLight jako child vaku** (ne sourozenec v group): při
  `animateBalloonBob` se světlo pohupuje spolu s vakem → lantern efekt
  zůstává centrovaný. Scale envelope.y = 1.15 na light.position (0,0,0)
  nemá vliv (scale × 0 = 0).
- **Druhá shadow mapa** (kromě slunce) = castShadow PointLight: cube
  shadow camera = 6 render pass. mapSize 512² drží perf přijatelný pro
  „občas svítící" lantern.
- **Click event místo pointerdown:** eliminuje konflikt s OrbitControls —
  `click` fire-uje jen při down+up bez drag, takže orbit-drag kamery
  přes balón LIT netoggle-uje.

Po ověření (vše napoprvé!) uživatel vybral **COUNTER** jako druhý
nevizuální potomek — demonstrace HUD observability. Dynamické vytvoření
řádku v `#hud` přes `createElement` + `textContent` (XSS-safe, NAME může
být v budoucnu user-input).

Pak ještě třetí feature: **edge highlight při hover** jako příprava pro
editor. `EdgesGeometry(geom, 20°)` + `LineSegments` s `depthTest=false` +
`renderOrder=999` → X-ray žluté hrany přes vše. Cache v `userData.edgeOverlays`
(lazy build, reuse). SPRITES skip (2D billboardy nemají 3D hrany).

### Schválená rozhodnutí

- **DD-17** — Diskrétní reakce na `TIME.tick`: TIMER + ACTION dispatch.
  `TIMER { INTERVAL, ACTION }` firuje `ACTION = { kind, target, attr, ... }`
  každých `INTERVAL` ticků. Engine dispatch `ACTIONS[kind]` (aktuálně
  `toggle`, `set`). Tick handler registry `tickHandlers[]` volaný
  z rozšířeného `setInterval` za `advanceTime()`. Registrace přes
  `registerBehavior(instance)` symetrický sibling `scene.add(createMeshFor(...))`
  pro vizuální entity. Watchery engine-derived behaviorů (DD-16 bubble
  tail, DD-17 LIT fade) per-frame reagují na stav v modelu. Více
  „ovladačů" téhož stavu (TIMER + click → `LIT`) je legitimní.

### Kód

**Model (`src/model.js`):**

- `BALLOON` dostal atribut `LIT = false` (bool, default).
- `class TIMER extends OBJECTS` — atributy `INTERVAL` (int) a
  `ACTION = { kind, target, attr, value? }` (default null).
- `class COUNTER extends OBJECTS` — atributy `VALUE` (int, default 0)
  a `INCREMENT` (int, default 1, může být záporné).

**Engine (`src/main.js`):**

- Import rozšířen o `TIMER`, `COUNTER`, `TIME`.
- `tickHandlers[]` registry + `updateTickHandlers()` volaný ze
  `setInterval` callback (po `advanceTime`).
- `ACTIONS = { toggle, set }` dispatch mapa + `dispatchAction(action)`.
- `registerTimer(instance)` — closure s `lastFire = TIME.tick`, fire
  při `tick - lastFire >= INTERVAL`.
- `registerCounter(instance)` — dynamicky přidá HUD řádek (createElement
  + textContent), closure mutuje `VALUE += INCREMENT` a updatuje DOM.
- `registerBehavior(instance)` — dispatch TIMER / COUNTER.
- `litEntities[]` + `registerLit(instance, envelope, light)` + `updateLit(dt)`
  — exp. lerp `emissive` + `light.intensity` (factor `1 − exp(−5·dt)`,
  ~0.5 s fade). Konstanty `LIT_MAX_EMISSIVE = 1.5`, `LIT_MAX_LIGHT = 10`.
- `buildBalloon` rozšířen: `envelopeMat` dostal `emissive` (vlastní COLOR)
  + `emissiveIntensity = 0`; nový `PointLight(0xffb060, 0, 12, 2)` jako
  child envelope s castShadow + shadow.mapSize 512² + near/far
  0.1/15. `registerLit(instance, envelope, light)` na konci.
- `canvas.addEventListener("click", ...)` — raycaster na pointer, pokud
  hit `BALLOON` instance → `instance.LIT = !instance.LIT`.
- Render loop spočítá `dt` z `performance.now()` delta, volá `updateLit(dt)`
  za `updateBubbleTails()`.
- `setEdgeHighlight(instance, on)` — lazy build `EdgesGeometry` per mesh
  child, `LineSegments` s `color 0xffff00`, `depthTest=false`,
  `transparent=true`, `renderOrder=999`, `raycast=()=>{}`. Cache v
  `root.userData.edgeOverlays`. SPRITES skip.
- Pointermove handler track-uje `lastHoveredInstance`; při přechodu
  vypne starý overlay a zapne nový. Pointerleave vypne vše.

**Instance:**

- `timer_0001` „Timer lampionu" INTERVAL=5, ACTION toggle balloon1.LIT.
- `counter_0001` „Skóre" start=0 inc=1 — HUD řádek pod TIME.

**HTML/CSS (`index.html`):**

- `#hud > div { margin-top: 2px; }` — drobný rozestup pro dynamické
  COUNTER řádky pod TIME.

**Audit code findings F1–F6 (sez. 8)** — všech 6 opraveno ve stejném
sezení 8: `SPEAKER_OFFSET_Y` atribut (F1), DD-16 + GLOSSARY (F2),
clarifying komentáře (F3/F4), sdílené materiály v `buildRock` (F5),
`userData.noShadow` flag pro bubble tail (F6).

**Docs:**

- `docs/DESIGN_DECISIONS.md` — DD-17 přidáno.
- `docs/GLOSSARY.md` — BALLOON rozšířen o LIT/lantern; TIMER, COUNTER,
  ACTION zaneseny; TIME popis aktualizován.
- `docs/TODO.md` — dva M8+ body odškrtnuty (TIMER/COUNTER, lantern +
  edge highlight jako bonus).

## Sezení 10 — %AUDIT:CODE fixy, CHARACTER + mode system, wander automat, 2D kolize

### Diskuse

Sezení otevřeno `%AUDIT:CODE` po DD-17 (plnění Příště sez. 9 bodu 4).
Audit identifikoval 6 findings (F1–F6); uživatel řekl „opravit vše",
opraveno F1–F4 (F5/F6 jako INFO, jeden „no change needed").

Hlavní obrat: **„Konečně animovaná postavička?"** Po krátkém
design-recommendation cyklu (kostka vs. organické, kolem čeho chodí)
uživatel stanovil požadavky:

1. Ruce i nohy ze **dvou dílů** s kulovým kloubem uprostřed; klid = loutka.
2. Tři side-requesty ve stejné zprávě: **zvýšit světlo rozsvíceného balónu**
   (+dosah, ostřejší stíny), **edge highlight jen viditelné hrany** (depthTest
   true, polygonOffset), **torzo jako válec** ne kvádr + **spodní díly s větší
   amplitudou** (stylizované overlapping action).

Poté: **„postavy musejí mít módy: chůze, běh, stoj, sed, leh, zvedání,
pokládání, …"**. Zásadní design rozhodnutí:

- **ANIMATE slot rozšiřujeme z „plynulé animace" na „mode slot".** Statické
  pózy (`sit`, `lie`) jsou animátory, co neignorují `t` a jen nastaví rotace;
  dynamické (`walk`, `run`) čtou `t`. Jeden dispatch, jeden mentální model.
- „Zvedání/pokládání" jsou **sekvence** (pre-pose → mid-pose → post-pose +
  interakce s objektem) — nespadají do módu, budou vyžadovat samostatný návrh.

User pak: **„Cílem je automat, postavy se poflakují."** Implementace
stavového automatu **uvnitř animátoru** (`wander` kind, state v
`group.userData.wander`) — 6 stavů (walk/run/stand/sit/lie/work), work je
dvoufázový (approach → perform). Sdílené pose primitives (`applyWalkCycle`,
`applySitPose`, `applyLiePose`, `applyWorkPose`, `resetCharBase`,
`moveTowards`) vytaženy pro reuse mezi samostatnými animátory (`walk`,
`sit`) a wander.

Finální: **kolizní systém**. 2D kruhy v XZ, dispatch-by-type radii, stop &
transition (žádný sliding/pathfinding). Pohyblivé entity (characters,
orbiting box) čtou live pozici přes `object3d.position`.

**Kuriozita sezení:** User jednou vloupil prompt z jiného projektu („přidáme
další --pdf bool, ZIP s fakturami SK") — AI to flag-ovalo jako přehození okna
místo pokusu implementovat nesmysl.

### Schválená rozhodnutí

- **DD-18** — CHARACTER třída s dvoudílnými končetinami a ANIMATE jako
  „mode slot" (statické + dynamické pod jedním dispatchem). `wander` jako
  stavový automat uvnitř animátoru.
- **DD-19** — 2D kolizní systém v XZ; dispatch-by-type radii; stop &
  transition (bez slidingu/pathfindingu).

### Kód

**%AUDIT:CODE fixes (main.js):**

- **F1:** `registerLit` přesunut z `buildBalloon` do `createMeshFor` —
  engine-derived registrace na jedné úrovni detailu (vedle `registerAnimator`
  + `meshByInstance.set`).
- **F2:** Odstraněn komentář tlačící k předčasnému `updaters[]` refaktoru
  (N=3 registries se specifickými signaturami neospravedlňuje abstrakci).
- **F3:** `registerCounter` handler má uniform signaturu `(_tick) => void`
  s `registerTimer`.
- **F4:** Cleanup poznámka konsolidována u `meshByInstance` na umbrella
  komentář pokrývající všech 6 monotonních registrů.

**Model (`src/model.js`):**

- `class CHARACTER extends COMPOSITES` s atributem `COLOR`. Dokumentace
  popisuje kostru (torso cylinder, 4 dvoudílné končetiny, kloubové pivoty).

**Engine (`src/main.js`):**

- `buildCharacter(group, instance)` — torzo (CylinderGeometry konická 0.19→0.15),
  hlava (SphereGeometry), 4 končetiny přes pomocný `buildLimb()` helper (horní
  válec + kloub koule + sub-Group pro spodní díl). Reference v
  `group.userData.parts`.
- Dispatch v `createCompositeFor` rozšířen o `CHARACTER`.
- Import CHARACTER z modelu.
- **Edge highlight** (DD-19 nesouvisející) — `LineBasicMaterial` teď
  `depthTest=true`, `polygonOffset=true/-2/-2` (jen viditelné hrany,
  prevence z-fight proti povrchu). `renderOrder` = default.

- **Pose primitives** (sdílené): `applyWalkCycle(p, t, period, amp, lowerFactor=0.6)`,

  (moveTowards subject.X/Z stopDist=1.1) → perform (3–5 s work pose).
- **`ANIMATORS`** rozšířeny o `walk`, `sit`, `wander`.
- **Collision systém:** `collidables[]`, `collisionRadiusFor(instance)`
  (dispatch CHARACTER 0.2, TREE 0.35, HOUSE 0.85, ROCK/voxel 0.5, floating
  null), `registerCollisionFor` v `createMeshFor`, `isBlocked(moving, nx, nz)`
  lineární check. `moveTowards` checkne před posunem, při kolizi return `true`
