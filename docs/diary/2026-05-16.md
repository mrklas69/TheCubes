### Rozhodnutí

- **`RESOURCE_YIELD` lookup tabulka** (`src/composites/builders.js`, export per KIND): spruce=8 wood, oak=12 wood, palm=6 wood, rock=4 stone, stump=3 wood, log=5 wood, cactus=2 wood, bush/flower/grass_tuft={}. Driving spec = AI default sez. 52 návrh + user Go acceptance. Cactus jako wood proxy (botanicky kaktus dřevo nemá, ale 5 surovin per DD-57, "plant fiber" neexistuje → wood nejbližší abstrakce).
- **`hasChopYield(kind)`** predicate — sum > 0 = chopable, jinak vegetace bez akce.
- **Lazy chop bucket pattern** — abstract `CUBES` instance 1 cell nad top TCUBES, lazy-create při prvním chop na column, drží v `_chopBucketByColumn: Map<"x,z", CUBES>`. Bucket je v `_voxelOwners` Set pro `rebuildAllVoxels()` replay.
- **`rebuildAllVoxels()` full re-dispatch** — chop = O(all voxels) přepočet, ne incremental. Per-voxel batch tracking je sub-prah (20×20 target use case max ~40 scatter + few owners = <1 ms). Owners replay + scatter replay z `_lastSpawnArgs` cache (deterministic per seed = stejný pattern).
- **BFS overflow** — 4-neighbor (N/S/E/W), max 3 hops, silent drop po exhaust (A11 emise zahodit varianta).
- **Hover diferenciace** — `HOVER_EMISSIVE_CHOP = 0x802020` (warm red) pro chopable DECOR, `HOVER_EMISSIVE_HEX = 0x404020` (žlutá) jinak. Plus cursor cue `canvas.style.cursor = "pointer"` nad chopable, default jinde.
- **Žádný nový DD** — DD-49 spec extension (RESOURCE_YIELD), bucket pattern = implementace pod DD-56 koncept 11 umbrellou. DD log immutable, ne novum.
