**F1 (kritický) — Y konvence dluh.** Sez. 17 měl tři konvence pro
`instance.Y` napříč potomky CUBES:
- BLOCKS (TCUBES, TRRAMPS, TTRAMPS, TTUNELS) → grid Y voxelu (= mesh
  center, snap-to-int v rendereru)
- VOXEL_MODEL → world Y surface (= mesh bottom, auto-snap loaderu)
- Pixel-voxel COMPOSITES (TREE, GRASS_TUFT, ROCK_PIXEL, LOG) → surface
  + 0.5 (= group origin, kvůli `treeVoxel` lokálnímu offsetu `-0.5 +
  (gy + 0.5) * TREE_PX`)

User v sez. 17 narazil na bug („stromy utopené v podlaze") právě kvůli
tomu — DD-22 vzorec `t.y + 0.5` byl chybný pro pixel-voxel; populate
musela použít `t.y + 1`.

**Tři možné fixy (audit findings):**
- **A)** Sjednotit pixel-voxel + VOXEL_MODEL na **surface** (mesh bottom).
  `treeVoxel` zahodí `-0.5 +` offset → group origin = surface.
- **B)** Sjednotit vše na grid-Y (mesh center). Náročnější, ovlivní
  auto-center bottom-snap loaderu.
- **C)** Akceptovat tři konvence + nový DD-28 popisující per-type
  semantics + tabulka v GLOSSARY.

User implicitně přijal A → implementováno. **DD-28** zafixoval **dvě**
sémantiky (ne tři): BLOCKS = grid-center, surface třídy = mesh bottom.
